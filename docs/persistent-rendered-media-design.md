# QQ 撤回媒体快照跨重启设计

## 目标

为 QQ 9.9.32-51246 中缺少有效本地源文件的图片表情和商城表情保存本地媒体副本，使其在退出并重新启动 QQ 后仍可恢复。优先保留原动画；原始字节不可读时允许降级为静态 PNG；全程不访问外部网络。

## 范围

- 只处理当前插件已经支持、且被标记为 `memoryOnly` 的图片或表情元素。
- 保持文字、回复、原生小表情、有效本地图片和有效本地商城表情的现有行为。
- 语音、视频、文件、转发记录和复杂卡片继续不处理。
- 不创建或更新朋友分发包。
- QQ.exe 与 application.asar 保持不变。

## 方案比较与选择

### 方案 A：只序列化 DOM

保存节点 HTML 或 `src` 字符串。实现最小，但 `blob:` 生命周期随渲染进程结束，不能满足跨重启要求，因此不采用。

### 方案 B：统一 Canvas PNG

所有媒体绘制到 Canvas 后保存 PNG。跨重启稳定，但动画表情只剩静态首帧，因此只作为降级路径。

### 方案 C：原始字节优先，Canvas PNG 降级

先核实 QQ 实际节点的 `src/currentSrc`、协议、MIME 和可读性。仅对本地来源读取原始字节并保留 GIF/WebP 等动画；原始字节不可读但 Canvas 可绘制时保存 PNG。该方案满足动画优先和稳定降级要求，作为正式设计。

## 数据流

1. 渲染层在撤回前继续保存最多 500 条 DOM 快照，并读取目标媒体节点的 `currentSrc/src`、MIME、尺寸和动画候选信息。
2. 捕获入口只接受 `data:`、`blob:` 或经实机确认属于 QQ 本机资源的固定协议；`http:`、`https:` 及协议相对地址立即拒绝。
3. 原始字节可读时保持原 MIME；不可读时尝试 Canvas 导出 `image/png`。捕获结果限制为单项不超过 20 MiB。
4. 渲染层通过固定 IPC 发送 `{ messageId, mimeType, bytes }`，不接受任意输出路径。
5. 主进程计算 SHA-256，以 `media/<sha256>.<ext>` 原子写入当前记录根目录，并返回只含哈希、相对路径、MIME、大小和降级标记的媒体引用。
6. 处理器将媒体引用附加到对应撤回记录；纯媒体 `memoryOnly` 消息在媒体落盘成功后允许持久化，混合消息在原有文字记录上补充媒体引用。
7. QQ 重启后，处理器从记录恢复媒体引用；渲染层通过固定 IPC 读取按哈希校验过的媒体字节，创建页面内可显示的本地 URL，并在节点销毁后释放临时 URL。

## 存储格式

- 媒体目录：`<recordRoot>/media/`，本机当前配置对应 `G:\QQ\media`。
- 文件名：小写 SHA-256 加固定扩展名；扩展名只由受支持 MIME 映射生成，不使用消息输入文件名。
- 记录引用：`{ sha256, relativePath, mimeType, sizeBytes, staticFallback }`。
- 相对路径必须匹配 `media/<64位小写十六进制>.<固定扩展名>`，解析后的绝对路径必须仍位于当前记录根目录。
- 相同内容按 SHA-256 去重，不重复写文件。

## 清理与一致性

- 保存采用同目录临时文件加原子重命名；哈希或大小不一致时不写记录。
- 加载时重新校验路径范围、文件大小和 SHA-256；失败时忽略媒体引用，保留可恢复的文字部分。
- 删除会话后扫描剩余记录引用，只删除已经没有任何记录引用的媒体文件。
- 修改记录目录时复制记录 JSON 和仍被引用的媒体文件；原目录继续保持不自动删除的现有规则。
- 不增加自动容量淘汰策略，媒体空间由现有按会话删除功能管理，避免后台误删仍被引用的内容。

## 安全边界

- 生产源码不得访问 `http:` 或 `https:`，不得下载或补取远端资源。
- IPC 只传消息 ID、受限 MIME、有限大小字节和内容寻址引用；主进程不接受渲染层传入任意文件路径。
- 单项 20 MiB 上限、固定 MIME 白名单和 SHA-256 校验同时应用于写入与读取。
- 临时实机诊断只记录协议、MIME、尺寸和可读性，不记录或上传用户媒体内容；诊断代码不进入仓库和交付包。

## 错误处理

- 原始字节不可读：尝试 PNG 静态降级。
- Canvas 被污染或导出失败：保留当前会话 `memoryOnly` 行为，不生成损坏文件。
- 文件写入失败：不改变原记录，不影响文字恢复。
- 文件丢失或校验失败：跳过媒体，继续恢复其他有效元素，并保留可诊断错误信息。

## 测试与验收

- TDD 覆盖原始动画字节保存、PNG 降级、协议拒绝、MIME 白名单、20 MiB 上限、SHA-256 去重、原子写入、损坏文件、路径逃逸和孤儿媒体清理。
- 覆盖纯表情与文字加表情的缓存、撤回恢复、持久化、QQ 重启后恢复及记录删除。
- 保留文字、普通图片、操作者提示、相同提示不重复写 DOM、当前会话切换和缓存淘汰回归。
- 静态审核确认无 `http:`、`https:`、下载、子进程、动态执行和任意路径 IPC。
- 包校验、部署前备份、安装源码一致、记录文件不丢失、QQ 进程响应及 QQ.exe/application.asar 固定哈希全部通过。
- 实机最终动作：另一账号发送并撤回动画表情，确认当下保留；退出全部 QQ 进程并重启，确认动画或静态降级图仍显示；删除对应测试会话记录后确认引用媒体被清理且其他记录不受影响。

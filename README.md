# QQ 本地防撤回部署

面向 Windows x64 QQ `9.9.32-51246` 的 LiteLoaderQQNT 插件。插件只处理本机已经收到的消息，不修改 QQ 服务器状态或网络协议。

## 功能

- 保留文字、QQ 原生小表情、回复、混合消息，以及普通图片、图片/动画表情和商城表情；优先复用有效本地文件，当前会话资源未落盘或已失真时复用撤回前的渲染快照。
- 撤回提示显示真实操作者；普通成员显示“成员名 尝试撤回…”，管理员或群主撤回他人消息时同时显示角色、操作者和原发送者。
- 已撤回记录跨 QQ 重启保留。
- 按好友或群聊显示记录数、实际文件占用和最后撤回时间。
- 好友缺少昵称时显示可读的“好友（QQ号）”，不直接展示 QQ 内部 UID。
- 管理页面显示当前记录目录，支持选择新的本机磁盘目录并迁移已有记录。
- 按占用降序排列，支持搜索、勾选并整组删除。
- 默认不阻止自己撤回消息。
- 插件源码没有外部网络访问、第三方运行时依赖或原生模块。

## 不支持

- 已重启且本机图片/表情资源均缺失的内容，以及语音、视频、文件、转发记录和复杂卡片恢复。
- 插件安装前、QQ 离线期间已经完成的撤回。
- QQ 更新到新版本后的自动适配。

## 使用

1. 完全退出 QQ。
2. 以 PowerShell 运行交付目录中的 `install.ps1`。
3. 启动 QQ，在“设置 > QQ 本地防撤回”中打开记录管理。
4. 如需恢复原状，完全退出 QQ 后运行 `rollback.ps1`。

详细说明见 [docs/installation.md](docs/installation.md)。

记录保存位置可在“管理记录”窗口中查看并修改；安装脚本也支持自动查找 QQ 目录。图片和表情只复用 QQ 已落盘的有效缓存或当前页面已经显示的渲染快照，插件不会下载或补取远端资源；仅靠渲染快照恢复的内容不写入跨重启记录。
